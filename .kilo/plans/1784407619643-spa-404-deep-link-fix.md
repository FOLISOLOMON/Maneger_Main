# Fix: 404 on refresh of deep links (Netlify + Vercel)

## Diagnosis (confirmed)
- `vite dev` / `vite preview` already do SPA fallback → deep links return 200. Not a
  build/code bug.
- Symptom occurs in a **plain browser tab at the deployed URL** (not the installed
  PWA), on Netlify or Vercel. The host serves raw static files and returns a real 404
  for paths that don't exist as files (`/inventory`, `/sales`, `/settings`, ...).
- The service worker's `navigateFallback: '/index.html'` only helps *after* the PWA is
  installed and the SW controls the page; it does not help a direct server request in a
  normal tab. So the server must rewrite unknown routes to `index.html`.

## Fix
Add SPA fallback config that Vite copies from `public/` into `dist/` at build time.

### 1. Netlify — `public/_redirects`
```
/*    /index.html   200
```
This rewrites every unmatched path to the app shell.

### 2. Vercel — `public/vercel.json`
```json
{
  "rewrites": [{ "source": "/(.*)", "destination": "/index.html" }]
}
```
(Vercel reads `vercel.json` from the project root; placing it in `public/` also works
and guarantees it ships with the build output. If a root `vercel.json` already exists,
merge the `rewrites` array instead of adding a second file.)

## Files touched
- `public/_redirects` (new)
- `public/vercel.json` (new)

## Validation
1. `npm run build` — confirm `dist/_redirects` and `dist/vercel.json` exist.
2. For Netlify:
   - `npx netlify dev` (or deploy preview) then load `/inventory`, refresh → 200,
     app renders the Inventory page (not 404).
3. For Vercel:
   - `npx vercel dev` then load `/sales`, refresh → 200, Sales page renders.
4. `npm run lint` / `npm run typecheck` unaffected (no src changes).

## Notes
- No changes to `vite.config.ts` or app code are needed; the existing
  `navigateFallback: '/index.html'` remains for installed-PWA offline support.
- If a root-level `vercel.json` already exists in the repo, do NOT create a second one;
  instead add the `rewrites` rule to it.
